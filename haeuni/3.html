3. WEB HOSTING

클라이언트가 웹에 대한 접근을 request하면, 해당 웹을 관리하는 서버는 클라이언트의 request에 response하여 웹 접근을 허용한다.
웹 사이트를 사용자에게 제공하기 위해서는 웹 서버가 필수적인데, 개인 또는 기업이 직접 컴퓨터에 서버를 다운받아 웹 사이트를 제공하는 것은 까다롭다.
따라서 사용자에게 웹을 제공하려고 하는 많은 사람들은 웹 서버를 대신 제공해주는 web hosting 서비스를 이용하고 있다.
현재 그 중 정말 유명하고 많이 쓰이는 web hosting 제공 사이트로 'GitHub'가 있다. 'GitHub'는 무료 서비스이기 때문에 비용부담도 없다는 장점이 있다.

*GitHub 무료 web hosting 이용 방법*
1. GitHub에 접속한다.
2. file을 저장할 새 repository를 생성한다. (README 생성 check!)
3. 생성한 repository에서 'Add file' 클릭 - 'Upload files' 클릭 - web hosting을 맡기고 싶은 files 첨부 - 'Commit changes' 클릭
4. 'Settings' 클릭 - 'Options' 카테고리 클릭 - 'GitHub Pages' 항목에서 'none'이 아닌 ____ 선택
5. 부여받은 주소 배포
